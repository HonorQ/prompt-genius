# 【Code with SOLO】用 SOLO 半小时打造小白专属的 AI 生图“盲盒”向导工具

> *(可以在这里插入一张首页截图，展示炫酷的深色玻璃态风格)*

## 1. 摘要
本项目使用 TRAE SOLO 开发了一款名为 **PromptGenius** 的 Web 应用。它解决了普通小白用户在使用豆包、Midjourney 等生图工具时“不知道怎么写提示词”的痛点。通过提供全中文大白话的分步向导和“一键盲盒”功能，让没有任何基础的用户也能轻松生成结构化、高质量的专业提示词，大幅降低了 AI 绘画的使用门槛。

## 2. 背景
我是一名全栈开发者。随着 AI 绘画工具（如豆包、元宝、Stable Diffusion 等）的普及，我发现身边很多非技术背景的朋友（比如运营、策划甚至长辈）对 AI 绘画很感兴趣，但总是卡在“写提示词”这一步。他们通常只会写“画一只猫”，导致生成的图片效果很差；而网上的专业提示词教程又充满了晦涩的英文参数，让人望而却步。

因此，我希望利用 TRAE SOLO 的能力，快速搭建一个**真正面向小白用户的提示词向导工具**，将复杂的参数选项转化为直白的选择题，甚至提供“一键盲盒”兜底方案。

## 3. 实践过程

### 3.1 任务拆解
我将整个应用的开发拆解为以下几个核心步骤交由 SOLO 执行：
1. **基础框架搭建**：使用 React + Vite + Tailwind CSS 初始化项目。
2. **状态管理设计**：使用 Zustand 存储用户在各个步骤中选择的参数（主体、动作、画风、色彩等）。
3. **数据字典构建**：定义一个庞大的选项库，包含中英文双语对照（如：`'3d render, octane render|3D渲染，辛烷渲染'`）。
4. **向导 UI 开发**：实现分步表单（Wizard），并加入平滑的动画过渡（Framer Motion）。
5. **提示词拼接逻辑**：根据用户的选择，智能拼接出最终的中/英文长文本提示词。
6. **体验优化**：增加“一键盲盒”（完全随机参数）、一键复制（含 Fallback 机制）以及优秀案例展示。

### 3.2 关键操作与 SOLO 的表现
在整个开发过程中，SOLO 展现了极其强大的上下文理解和需求转化能力：
- **从“术语”到“大白话”的转化**：当我提出“需要更小白易懂的语言”时，SOLO 主动将枯燥的“设置光影与视角”改写成了“怎么打光？从哪拍？”，并为每个选项配上了生动的 Emoji 和通俗的解释（比如把“广角镜头”解释为“拍出很大的风景，人在画面里比较小”）。
- **复杂的随机逻辑实现**：在开发“一键盲盒”功能时，SOLO 不仅写出了随机抽取逻辑，还聪明地加入了 50% 的触发概率（比如场景和天气），避免最终生成的提示词过于拥挤矛盾。
- **完善的兜底机制**：当我反馈“一键复制按钮在某些环境下无效”时，SOLO 迅速定位到是因为非 HTTPS 环境下 `navigator.clipboard` 受到限制，并主动编写了基于 `document.execCommand('copy')` 的完美 Fallback 降级方案。

### 3.3 踩过的坑
在测试复制功能时，遇到了上述提到的本地预览环境 Clipboard API 权限问题。得益于 SOLO 的快速排查，我们在极短的时间内补充了传统的 textarea 复制逻辑，保证了功能在任何环境下的可用性。

## 5. 效果与总结
- **提效显著**：原本需要我自己构思 UI、寻找图标、编写复杂的选项字典并处理组件状态，预计需要花费 1-2 天的时间。而在 TRAE SOLO 的帮助下，整个应用从零到上线级别仅用了不到半小时！
- **AI 的进阶用法**：SOLO 不仅仅是一个“代码生成器”，它在这个项目中更像是一个**产品经理兼交互设计师**。它能够准确理解我“面向小白用户”的产品定位，并在文案、UI 细节和交互逻辑（如增加“跳过”按钮、参数回顾面板）上主动给出极佳的优化方案。
- **经验复用**：通过这次实践，我发现**“先搭骨架，再填血肉，最后磨皮（优化体验）”**是使用 SOLO 最舒适的节奏。先让 SOLO 跑通核心逻辑，然后再通过多轮对话不断细化和润色，能极大发挥它的潜力。
